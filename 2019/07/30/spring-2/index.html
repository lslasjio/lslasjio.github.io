何为“容器”

在学习该部分内容时，不妨先了解一下，在Java中，到底什么是“容器“。在百度百科中，对于Java容器，是这么介绍的：

​ 容器可以管理对象的生命周期、对象与对象之间的依赖关系，您可以使用一个配置文件（通常是XML），在上面定义好对象的名称、如何产生（Prototype 方式或Singleton 方式）、哪个对象产生之后必须设定成为某个对象的属性等，在启动容器之后，所有的对象都可以直接取用，不用编写任何一行程序代码来产生对象，或是建立对象与对象之间的依赖关系。

在Java中，“容器”可以看做是一种用来存储数据的数据结构，比如“集合”（Set）、“列表”(List)、“映射”(Map)等，此处内容的深入讨论，之后摸鱼时可再另开一章，先附上参考用的博客文章：CSDN|《JAVA中关于”容器“的理解》|作者：追梦111

Java反射机制

（Java反射机制参考文章CSDN|《Java反射机制》|作者：①块腹肌）

一、什么是反射机制

简单来说，反射机制就是指一个程序在运行的时候，能够获取自身的信息。在Java中，一般只要给一个类定义了名字，就可以通过反射机制获取该类的所有信息。

反射的概念是由Smith在1982年首次提出的，主要是指程序可以访问、检测和修改它本身状态或行为的一种能力。在计算机科学领域，反射特指能够进行自描述和自控制的一类应用。

（注：此处对该概念的提出者进行了查找，但仅查到了两个信息：Smith，1982，待完善）

这一操作，就是反射机制的使用，大部分Java开源框架就是使用反射机制进行实现的。

Java反射机制主要提供以下功能：

  • 在程序运行期间判断一个对象属于哪个类。
  • 在程序运行期间构造出任意一个类的对象。
  • 在程序运行期间判断任意一个类具有的成员变量和成员方法。
  • 在程序运行期间调用任意一个对象的方法。
  • 生成动态代理。

三、反射机制的优缺点

提到反射机制的优缺点，需要先简单提到Java中的两种编译方式：静态编译和动态编译

（两种编译方式参考文章：CSDN|《Java之动态编译，静态编译简单理解和实例》|作者：阿_毅）

  • 静态编译：在编译时把所有模块都编译进入可执行文件中，当该文件启动时，所有模块将被加载进来。即在编译时确定类型和绑定对象。
  • 动态编译：按需编译。程序在运行的时候，用到那个模块就编译哪个模块。

相较于两种方式，后者最大限度发挥了Java的灵活性，降低了类之间的耦合性（当然也有缺点，在上文所引用的文章中有提及）。

​ 对于反射机制而言，其优点就是能实现动态创建对象和编译，体现出很大的灵活性。比如，一个大型的软件，不可能一次就把把它设计的很完美，当这个程序编译后，发布了，当发现需要更新某些功能时，我们不可能要用户把以前的卸载，再重新安装新的版本，假如 这样的话，这个软件肯定是没有多少人用的。采用静态的话，需要把整个程序重新编译一次才可以实现功能 的更新，而采用反射机制的话，它就可以不用卸载，只需要在运行时才动态的创建和编译，就可以实现该功能。

它的缺点是对性能有影响。使用反射基本上是一种解释操作，我们可以告诉JVM，我们希望做什么并且它满足我们的要求。这类操作总是慢于只直接执行相同的操作。

关于JDOM

在关于容器的定义中，提到了这么一句话：

“您可以使用一个配置文件（通常是XML），在上面定义好对象的名称、如何产生（Prototype 方式或Singleton 方式）、哪个对象产生之后必须设定成为某个对象的属性等，”

对于使用IoC容器的Spring，自然也会使用XML文件进行配置，那么也就可以引申出JDOM了。

一、什么是JDOM

简单来说，JDOM就是Java中一种用来更加方便操作XML文件的工具。在百度百科中，是这样定义的：

JDOM是一种使用 XML（标准通用标记语言下的一个子集） 的独特 Java 工具包，它的设计包含 Java 语言的语法乃至语义。

JDOM是一个开源项目，基于树形结构利用了纯Java技术实现了对XML文档的解析，生成，序列化及多种操作。JDOM直接为Java编程服务，将SAX和DOM功能有效的结合起来。在使用和设计上尽可能地隐藏了XML文档使用过程中的复杂性。

好家伙，又提到了两玩意儿，一 SAX,一 DOM。在这里附上百科里的定义，仅做粗糙了解，具体的之后再开坑（看吧）。

  • SAX: SAX（simple API for XML）是一种XML解析的替代方法。相比于DOM，SAX是一种速度更快，更有效的方法。它逐行扫描文档，一边扫描一边解析。而且相比于DOM，SAX可以在解析文档的任意时刻停止解析，但任何事物都有其相反的一面，对于SAX来说就是操作复杂。
  • DOM: 文档对象模型（Document Object Model，简称DOM），是W3C组织推荐的处理 可扩展标志语言的标准编程接口。在网页上，组织页面（或文档）的对象被组织在一个树形结构中，用来表示文档中对象的标准模型就称为DOM。
